    Quick Start

    This guide will help you deploy your first GenAI IDP solution in under 30 minutes. We'll deploy a basic document processing pipeline that can extract text from documents and analyze them using AI.

    What You'll Deploy

    This quick start creates:

    • S3 Bucket: For document storage and processing
    • Lambda Functions: For document processing logic
    • Textract Integration: For text extraction from documents
    • Bedrock Integration: For AI-powered document analysis
    • DynamoDB Table: For storing processing results
    • CloudWatch Logs: For monitoring and debugging

    Step 1: Choose Your Deployment

    Navigate to the basic pipeline example:

    cd examples/bedrock-llm-processor

    Step 2: Configure Variables

    Create a terraform.tfvars file with your configuration:

    # terraform.tfvars
project_name = "my-genai-idp"
environment  = "dev"
aws_region   = "us-east-1"

# Optional: Customize bucket names (must be globally unique)
document_bucket_name = "my-genai-idp-documents-dev-12345"
results_bucket_name  = "my-genai-idp-results-dev-12345"

# Bedrock model configuration
bedrock_model_id = "anthropic.claude-3-sonnet-20240229-v1:0"

# Tags
tags = {
  Project     = "GenAI-IDP"
  Environment = "Development"
  Owner       = "your-email@company.com"
}

    Bucket Naming

    S3 bucket names must be globally unique. Consider adding a random suffix or your initials to ensure uniqueness.

    Step 3: Review the Configuration

    Examine the main configuration file:

    cat main.tf

    The basic pipeline includes:

    module "processing_environment" {
  source = "../../modules/processing-environment"

  prefix = var.prefix
  region = var.region

  # S3 configuration
  enable_versioning = true
  enable_encryption = true

  tags = var.tags
}

module "bedrock_llm_processor" {
  source = "../../modules/processors/bedrock-llm-processor"

  prefix = var.prefix
  region = var.region

  # Connect to processing environment
  processing_environment = module.processing_environment

  # Bedrock configuration
  bedrock_model_id = "anthropic.claude-3-sonnet-20240229-v1:0"

  tags = var.tags
}

module "web_ui" {
  source = "../../modules/web-ui"

  prefix = var.prefix
  region = var.region

  # Connect to processing environment
  processing_environment = module.processing_environment

  # UI configuration
  admin_email = var.admin_email

  tags = var.tags
}

    Step 4: Deploy the Infrastructure

    Initialize and deploy:

    # Initialize Terraform
terraform init

# Review the deployment plan
terraform plan

# Deploy the infrastructure
terraform apply

    When prompted, type yes to confirm the deployment.

    Deployment Time

    The initial deployment typically takes 5-10 minutes to complete.

    Step 5: Verify the Deployment

    After deployment completes, verify the resources:

    # Check outputs
terraform output

# Verify S3 buckets
aws s3 ls | grep genai-idp

# Check Lambda functions
aws lambda list-functions --query 'Functions[?contains(FunctionName, `genai-idp`)].FunctionName'

    Step 6: Test the Pipeline

    Upload a Test Document

    Upload a sample document to test the pipeline:

    # Create a simple test document
echo "This is a test document for GenAI IDP processing." > test-document.txt

# Upload to the document bucket
aws s3 cp test-document.txt s3://$(terraform output -raw document_bucket_name)/

    Monitor Processing

    Watch the CloudWatch logs to see the processing:

    # Get the log group name
LOG_GROUP=$(terraform output -raw processing_log_group)

# Stream logs
aws logs tail $LOG_GROUP --follow

    Check Results

    After processing completes, check the results:

    # List processed results
aws s3 ls s3://$(terraform output -raw results_bucket_name)/

# Download and view results
aws s3 cp s3://$(terraform output -raw results_bucket_name)/processed/ . --recursive

    Step 7: Understanding the Outputs

    Your deployment provides several useful outputs:

    terraform output

    Key outputs include:

    • document_bucket_name: Where to upload documents for processing
    • results_bucket_name: Where processed results are stored
    • processing_log_group: CloudWatch log group for monitoring
    • api_endpoint: API Gateway endpoint (if enabled)

    Cleanup

    When you're done testing, clean up the resources to avoid charges:

    # Destroy all resources
terraform destroy

    Type yes when prompted to confirm the destruction.

    Data Loss

    This will permanently delete all resources and data. Make sure to backup any important documents or results before running destroy.

    Troubleshooting

    Common Issues

    Bucket Name Already Exists

    # Error: bucket name already exists
# Solution: Change bucket names in terraform.tfvars to be unique

    Insufficient Permissions

    # Error: Access denied
# Solution: Verify AWS credentials and permissions
aws sts get-caller-identity

    Bedrock Model Access

    # Error: Model access denied
# Solution: Request access to Bedrock models in AWS Console
